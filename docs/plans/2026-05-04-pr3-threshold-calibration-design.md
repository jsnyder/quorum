# PR 3: PR-Curve Threshold Calibration — Design

**Goal:** Replace hardcoded magic numbers in the calibrator (1.5, 2.0x ratios) with data-driven thresholds derived from the feedback corpus via precision-recall curve analysis.

**Architecture:** Pure Rust `src/metrics.rs` module implements PR curve + threshold selection. `quorum calibrate` CLI subcommand joins feedback labels with calibrator trace scores, runs the curve, and writes `~/.quorum/calibrator_thresholds.toml`. The calibrator reads thresholds at startup, falling back to legacy magic numbers if no file exists.

**Signals:** Precedent-only for this PR (`tp_weight / (tp_weight + fp_weight)`). LLM confidence and grounding status are additive modifiers for a follow-up PR.

---

## Components

### 1. `src/metrics.rs` — PR curve + threshold selection

- `precision_recall_curve(samples: &[(f64, bool)]) -> Vec<(f64, f64, f64)>` — sort by score desc, walk accumulating TP/FP, emit (precision, recall, threshold) at each distinct score.
- `threshold_at_precision(curve, min_precision) -> Option<f64>` — lowest threshold achieving target precision.
- `f1_optimal_threshold(curve) -> Option<f64>` — convenience for ablation.
- ~80 lines implementation, ~120 lines tests. Zero new crate dependencies.

### 2. `quorum calibrate` CLI subcommand

Pipeline:
1. Join `feedback.jsonl` (ground-truth: tp/partial → positive, fp → negative, skip wontfix) with `calibrator_traces.jsonl` (scores: `tp_weight`, `fp_weight`) on `finding_title + file_path`.
2. Score = `tp_weight / (tp_weight + fp_weight)`. Filter NaN (zero-weight entries).
3. Run PR curve on `(score, label)` pairs.
4. Pick `suppress_threshold` at precision 0.95, `boost_threshold` at precision 0.85.
5. Write `~/.quorum/calibrator_thresholds.toml`.

**Data quality gates:**
- Minimum 20 total labeled samples.
- Minimum 10 in minority class per path (suppress needs ≥10 FPs, boost needs ≥10 TPs).
- 10-30 minority samples: write thresholds but print confidence warning.
- Per-path independence: if boost has enough data but suppress doesn't, write only `[boost]`.

Output format:
```toml
# Generated by `quorum calibrate` on 2026-05-04
# Corpus: 1804 labeled samples (1346 TP, 458 FP)
[suppress]
precision_target = 0.95
threshold = 0.78

[boost]
precision_target = 0.85
threshold = 0.42
```

CLI prints: corpus size, class balance, thresholds, precision/recall at each operating point.

### 3. Calibrator runtime integration

`CalibratorConfig` gains:
```rust
pub suppress_threshold: Option<f64>,
pub boost_threshold: Option<f64>,
```

Read from `~/.quorum/calibrator_thresholds.toml` at init. If absent/malformed, `None` (legacy fallback).

Decision logic changes from magic-number ratios to:
```rust
let score = tp_weight / (tp_weight + fp_weight);
if let Some(thresh) = config.suppress_threshold {
    if score <= (1.0 - thresh) && fp_weight > 0.0 { suppress }
} else {
    // Legacy: fp_weight >= 1.5 && fp_weight > tp_weight * 2.0
}
```

Same pattern for boost. Legacy fallback = zero behavior change for users without thresholds file.

`QUORUM_FORCE_THRESHOLD=<float>` env var overrides both (ablation/sweeps).

### 4. Edge cases

- **Empty/tiny corpus (<20):** exit without writing, print message.
- **Degenerate class balance (<10 minority):** skip that path, print message.
- **Low minority count (10-30):** write but warn about instability.
- **No calibrator traces:** print "run a review with tracing first."
- **Malformed TOML:** `tracing::warn`, fall back to legacy. Never crash.
- **NaN scores (zero weight):** filter before curve computation.

## Test-first targets

1. PR curve on trivial 4-sample dataset — exact precision/recall/threshold values.
2. PR curve with tied scores — one point per distinct threshold.
3. `threshold_at_precision(0.95)` on known curve — correct threshold.
4. `threshold_at_precision(1.0)` when unachievable — returns `None`.
5. TOML round-trip: write thresholds, read back, values match.
6. `CalibratorConfig` with `suppress_threshold = Some(0.78)` changes behavior vs `None`.
7. `QUORUM_FORCE_THRESHOLD` overrides config file.
8. Legacy behavior unchanged when no TOML exists (regression guard).
9. Class balance gate: 25 TP + 2 FP → suppress path skipped.
10. Per-path independence: enough TPs but not FPs → only boost written.

## Ablation knobs

- `QUORUM_FORCE_THRESHOLD=<float>` — override both thresholds.
- Legacy fallback when no TOML file exists.
- `quorum calibrate --dry-run` — compute and print but don't write.
